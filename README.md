# on-m-hub

## 项目简介

`on-m-hub` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为静态 HTML 资源的集合与展示空间。  
所有页面均为独立编写，彼此之间无直接关联，旨在提供一份可浏览、可复用的 HTML 内容存档。

## 目录说明

```
on-m-hub/
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放独立 HTML 页面
│   ├── page-a.html
│   ├── page-b.html
│   └── ...
├── assets/             # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **pages/**：主要目录，每个 HTML 文件为一个独立页面，可单独访问。
- **assets/**：存放多个页面共用的资源文件，便于维护与复用。
- **index.html**：可选，可放置仓库导航或简介页面。

## 页面归档说明

本仓库中的 HTML 页面均以静态文件形式归档，内容可能涵盖各类主题（如工具、演示、信息展示等）。  
每个页面均为独立文件，不依赖外部数据库或后端服务，可直接在浏览器中打开使用。

归档目的包括：
- 保存特定时间点完成的 HTML 页面
- 提供无需服务器即可预览的网页资源
- 方便他人参考或二次开发

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面时会同步更新目录结构。
- 所有页面应遵循基本的 HTML 规范，并尽量保持代码简洁可读。
- 欢迎通过 Issue 或 Pull Request 提交建议或修正，但请注意本仓库不接收涉及具体域名、商业推广或诱导点击的内容。
- 对于长期未维护的页面，可能会被标记或归档至子目录中。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接打开 `pages/` 下任意 HTML 文件即可查看。
3. 也可通过 GitHub Pages 等服务部署在线访问（需自行配置）。
